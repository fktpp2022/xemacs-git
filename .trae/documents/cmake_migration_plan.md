# XEmacs Autoconf to CMake Migration Plan

## 1. Current Build System Analysis

### 1.1 Overview

The current XEmacs build system uses a **custom autoconf-based system** without automake. Key characteristics:

- **configure.ac**: ~50KB+ file with heavily customized XEmacs-specific macros
- **Makefile.in.in**: Manually crafted makefile templates (not generated by automake)
- **Installation**: Complex custom `make install` target
- **Feature detection**: Extensive checks for system features, libraries, and window systems

### 1.2 Key Directories and Files

| File/Directory | Purpose |
|----------------|---------|
| `configure.ac` | Main autoconf configuration |
| `Makefile.in.in` | Top-level makefile template |
| `src/Makefile.in.in` | Core XEmacs sources |
| `lib-src/Makefile.in.in` | Utility programs |
| `lwlib/Makefile.in.in` | Xt/Athena widget library |
| `modules/*/` | Dynamically loadable modules |
| `INSTALL` | Installation guide |
| `Installation` | Generated configuration summary |

### 1.3 Installation Directory Structure

Current layout (from `Makefile.in.in`):

```
${prefix}/
├── bin/                          # Executables (xemacs, etags, etc.)
├── share/
│   ├── man/                      # Man pages
│   └── xemacs-${version}/        # Version-specific data
│       ├── lisp/                 # Lisp libraries
│       ├── etc/                  # Architecture-independent data
│       └── info/                 # Info documentation
└── lib/
    └── xemacs-${version}/
        └── ${configuration}/     # Architecture-specific
            ├── DOC-*             # Documentation strings
            ├── config.values     # Configuration values
            └── modules/          # Dynamic modules
```

### 1.4 Major Feature Categories

The configure.ac detects:

1. **Compiler & Tools**: GCC, SunPro, DEC, Intel compilers; makeinfo, etc.
2. **OS & Architecture**: Linux, Solaris, AIX, Darwin, BSD, Windows (Cygwin/MinGW)
3. **Window Systems**: X11, GTK, Motif, Athena (Lucid), MS Windows
4. **Image Formats**: XPM, PNG, JPEG, TIFF, GIF, X-Face
5. **Sound**: Native OSS, ALSA, NAS, ESD
6. **Databases**: Berkeley DB, DBM/GDBM, LDAP, PostgreSQL
7. **Internationalization**: XIM, Canna, Wnn, fontconfig/Xft
8. **Networking**: Tooltalk, SOCKS, DECnet, IPv6, TLS/SSL
9. **Other Features**: Modules, relocating allocator, error checking, debug

## 2. Migration Strategy

### 2.1 Core Principles

1. **Use Modern CMake**: Target CMake 3.16+ for good feature support
2. **CMake Standard Practices**: Follow CMake conventions for `install()`, `find_package()`, etc.
3. **CPack Integration**: Use CPack for packaging instead of custom `make install`
4. **Preserve Functionality**: Maintain all existing configure options and behavior
5. **Incremental Migration**: Build in phases to allow testing

### 2.2 CMake Architecture

```
xemacs-git/
├── CMakeLists.txt              # Top-level
├── cmake/
│   ├── Modules/                # Custom find modules
```

## 3. Detailed Implementation Plan

### Phase 1: Core CMake Infrastructure

#### 3.1 Create Top-Level CMakeLists.txt

**Files to create:**
- `CMakeLists.txt` (top-level)
- `cmake/XEmacsOptions.cmake`

**Key elements:**
- Project declaration with version from `version.sh.in`
- Standard CMake setup (C standard, policies)
- Build options mirroring configure options:
  - `XEMACS_WITH_GTK`, `XEMACS_WITH_XFT`, etc.
  - `CMAKE_INSTALL_PREFIX` (standard CMake)
  - Build type: Debug/Release

#### 3.2 Version and Configuration Files

**Files to create:**
- `cmake/XEmacsConfigure.cmake`

**Tasks:**
- Parse `version.sh.in` to get version numbers
- Generate `src/config.h` from template (create `src/config.h.in.cmake`)
- Generate `src/paths.h` with installation paths
- Generate `Installation` equivalent (CMake configure summary)

**Configuration Summary (Installation equivalent):**
- Store in `CMakeCache.txt` (standard CMake)
- Additionally generate `Installation.txt` in build directory for user reference
- Use CMake's `message()` and `feature_summary()`

### Phase 2: Feature Detection System

#### 3.3 OS and Architecture Detection

**Files to create:**
- `cmake/XEmacsOS.cmake`

**Tasks:**
- Map autoconf's `$opsys` and `$machine` to CMake:
  - `CMAKE_SYSTEM_NAME` → opsys (Linux, Darwin, Windows, etc.)
  - `CMAKE_SYSTEM_PROCESSOR` → machine (x86_64, i386, etc.)
- Select appropriate `s/*.h` and `m/*.h` files
- Handle special cases (AIX, Solaris, Cygwin, MinGW)

#### 3.4 Compiler Detection

**Tasks:**
- Use CMake's `CMAKE_C_COMPILER_ID`
- Detect GCC, Clang, MSVC, Intel, SunPro, DEC C
- Set appropriate compiler flags:
  - Warning flags (`-Wall`, `-Wextra`, etc.)
  - Optimization flags (`-O3`, `-fno-strict-aliasing`)
  - Debug flags (`-g`)

#### 3.5 Find Modules for Dependencies

Create or use CMake find modules for:

| Library | CMake Module | Notes |
|---------|--------------|-------|
| X11 | `FindX11.cmake` | Built-in |
| GTK+ 2/3 | `FindGTK2.cmake`, `FindGTK3.cmake` | Built-in |
| Xft | Custom | Part of X11 but needs separate check |
| Fontconfig | Custom or `PkgConfig` | Use pkg-config if available |
| PNG | `FindPNG.cmake` | Built-in |
| JPEG | `FindJPEG.cmake` | Built-in |
| TIFF | `FindTIFF.cmake` | Built-in |
| GIF | Custom or `FindGIF.cmake` | CMake 3.18+ has FindGIF |
| XPM | Custom | Check for X11/Xpm.h |
| ALSA | `FindALSA.cmake` | Built-in |
| ncurses | `FindCurses.cmake` | Built-in |
| Berkeley DB | Custom | Check for db.h |
| GDBM | Custom | Check for gdbm.h |
| LDAP | `FindLDAP.cmake` | Built-in |
| PostgreSQL | `FindPostgreSQL.cmake` | Built-in |
| Canna | Custom | Japanese input method |
| Wnn | Custom | Japanese input method |
| TLS (OpenSSL/GnuTLS) | `FindOpenSSL.cmake`, `FindGnuTLS.cmake` | Built-in |
| zlib | `FindZLIB.cmake` | Built-in |
| GMP | `FindGMP.cmake` | CMake 3.13+ |

**Strategy for Custom Find Modules:**
1. Use `pkg_check_modules()` first if available
2. Fall back to `find_path()` and `find_library()`
3. Create `cmake/Modules/Find<Package>.cmake` for each

### Phase 3: Build Targets

#### 3.6 lib-src Utilities

**Directory:** `lib-src/`

**Programs to build:**
- `etags`, `ctags`
- `b2m`, `emacsclient`
- `gnuclient`, `gnudoit`, `gnuattach`
- `make-docfile`, `movemail`
- `hexl`, `ntlib.c` (Windows)
- And others...

**CMakeLists.txt elements:**
- Multiple `add_executable()` targets
- Proper dependencies
- Installation via `install(TARGETS ...)`

#### 3.7 lwlib Library

**Directory:** `lwlib/`

**Purpose:** Xt/Motif/Athena widget library for XEmacs

**Files:**
- `lwlib.c`, `lwlib-Xaw.c`, `lwlib-Xm.c`
- `xlwmenu.c`, `xlwscroll.c`

**CMakeLists.txt elements:**
- `add_library(lwlib STATIC ...)`
- Conditional compilation based on widget set (Lucid/Motif/Athena)
- Link with X11, Xt, Xmu, etc.

#### 3.8 Main XEmacs Binary

**Directory:** `src/`

**This is the most complex part** - requires careful analysis of `src/Makefile.in.in`

**Key aspects:**
1. **Source files**: ~150+ .c files
2. **Generated headers**:
   - `paths.h` (from paths.h.in)
   - `Emacs.ad.h` (from etc/Emacs.ad via ad2c)
   - `config.h` (from configure)
   - `symsinit.h` (generated)
   - `lisp.h` (includes many headers)
3. **Special build steps**:
   - Generating DOC file (documentation strings)
   - Dumping xemacs (creating `.dmp` file or embedding)
   - Byte-compiling Lisp files

**CMake strategy:**
- Use `add_executable(xemacs ...)`
- Use `configure_file()` for `paths.h`, `config.h`
- Use `add_custom_command()` for:
  - Generating `Emacs.ad.h` via `lib-src/ad2c`
  - Generating `symsinit.h`
  - Creating DOC file
  - Dumping process

#### 3.9 Dynamic Modules

**Directories:** `modules/*/`

**Modules:**
- `sample/` - Example module
- `ldap/` - LDAP support
- `postgresql/` - PostgreSQL support
- `canna/` - Canna input method
- `zlib/`, `base64/` - Various utilities

**CMake strategy:**
- Use `add_library(<module> MODULE ...)`
- Set `MODULE` target properties
- No `lib` prefix on module names
- Install to `${libdir}/modules/`

### Phase 4: Installation and Packaging

#### 3.10 Installation Rules

Use CMake's standard `install()` commands:

```cmake
# Binaries
install(TARGETS xemacs etags ctags ...
        RUNTIME DESTINATION bin)

# Libraries
install(TARGETS lwlib
        ARCHIVE DESTINATION lib)

# Lisp files
install(DIRECTORY lisp/
        DESTINATION share/xemacs-${version}/lisp
        FILES_MATCHING PATTERN "*.el" PATTERN "*.elc")

# Etc files
install(DIRECTORY etc/
        DESTINATION share/xemacs-${version}/etc)

# Info files
install(DIRECTORY info/
        DESTINATION share/xemacs-${version}/info)

# Architecture-dependent files
install(TARGETS modules
        LIBRARY DESTINATION lib/xemacs-${version}/${configuration}/modules)

# Man pages
install(FILES etc/xemacs.1 etc/etags.1 ...
        DESTINATION share/man/man1)
```

#### 3.11 CPack Configuration

**File:** `cmake/XEmacsCPack.cmake`

**Generators to support:**
- `TGZ` - Tar.gz
- `STGZ` - Self-extracting tar
- `DEB` - Debian packages (Linux)
- `RPM` - RPM packages (Linux)
- `NSIS` - Windows installer (if building for Windows)

**Key CPack variables:**
```cmake
set(CPACK_PACKAGE_NAME "xemacs")
set(CPACK_PACKAGE_VERSION "${EMACS_VERSION}")
set(CPACK_PACKAGE_DESCRIPTION_SUMMARY "XEmacs - The next generation Emacs editor")
set(CPACK_PACKAGE_VENDOR "XEmacs Development Team")
set(CPACK_RESOURCE_FILE_LICENSE "${CMAKE_SOURCE_DIR}/COPYING")
set(CPACK_RESOURCE_FILE_README "${CMAKE_SOURCE_DIR}/README")

# Debian-specific
set(CPACK_DEBIAN_PACKAGE_DEPENDS "libc6, libx11-6, ...")
set(CPACK_DEBIAN_PACKAGE_SECTION "editors")

# RPM-specific
set(CPACK_RPM_PACKAGE_REQUIRES "libX11, ...")
set(CPACK_RPM_PACKAGE_GROUP "Applications/Editors")
```

### Phase 5: Documentation

#### 3.12 Create CMake INSTALL Document

**File:** `INSTALL-CMAKE.md` (or `INSTALL` updated with CMake info)

**Content to include:**

1. **Basic CMake Build Instructions:**
   ```bash
   mkdir build && cd build
   cmake ..
   make -j4
   sudo make install
   ```

2. **CMake Options Equivalent to Configure:**
   | Autoconf Option | CMake Equivalent |
   |------------------|-------------------|
   | `--prefix=DIR` | `-DCMAKE_INSTALL_PREFIX=DIR` |
   | `--with-gtk` | `-DXEMACS_WITH_GTK=ON` |
   | `--with-xft` | `-DXEMACS_WITH_XFT=ON` |
   | `--without-x` | `-DXEMACS_WITH_X11=OFF` |
   | `--with-debug` | `-DCMAKE_BUILD_TYPE=Debug` |
   | `--enable-error-checking` | `-DXEMACS_ERROR_CHECKING=ON` |

3. **Available Build Options:**
   - Window system options (X11, GTK, MSW)
   - Image format options
   - Sound options
   - Database options
   - I18N options
   - Module options

4. **CPack Packaging:**
   ```bash
   cmake --build . --target package
   # Or
   cpack -G TGZ
   cpack -G DEB
   cpack -G RPM
   ```

5. **Out-of-Source Builds:**
   - Recommended CMake practice
   - Multiple configurations from same source

6. **Configuration Summary:**
   - Using `cmake -LA ..` to see all options
   - Using `ccmake ..` or `cmake-gui` for interactive configuration

## 4. Risk Assessment and Mitigation

### 4.1 Major Risks

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Complex feature detection missing edge cases | High | Critical | Test extensively on multiple platforms; reference autoconf tests exactly |
| Dump process not working correctly | High | High | The dump mechanism is complex; carefully analyze the current implementation |
| Platform-specific code paths broken | Medium | High | Need to handle all OSes (Linux, macOS, BSD, Solaris, AIX, Cygwin, MinGW) |
| Lisp byte-compilation order wrong | Medium | Medium | Follow exact order from src/Makefile.in.in |
| Paths compiled into binary incorrect | Medium | High | Verify paths.h generation matches original |

### 4.2 Mitigation Strategies

1. **Phased Development**: Build and test each component separately
2. **Comparison Testing**: Configure both systems and compare outputs:
   - `config.h` content
   - Compiler flags
   - Defined preprocessor symbols
3. **Platform Testing**: Test on at least:
   - Linux (x86_64)
   - macOS (if possible)
   - FreeBSD (if possible)
4. **Regression Testing**: Run existing test suite after migration

## 6. Success Criteria

The migration is complete when:

1. **CMake can configure XEmacs** with all major options
2. **Build completes** producing:
   - `xemacs` binary
   - All lib-src utilities
   - Compiled Lisp files (`.elc`)
   - DOC file
3. **Installation works** via `make install` with correct directory structure
4. **CPack generates valid packages** (TGZ, DEB/RPM where applicable)
5. **`INSTALL-CMAKE.md`** fully documents the CMake build process
6. **Configuration summary** (`Installation`) is generated in CMake way
7. **Existing tests pass** (if any)

## 8. Estimated Complexity

This is a **major migration effort** due to:

1. **Size of configure.ac**: ~50KB of custom autoconf macros
2. **Platform diversity**: Support for many Unix variants plus Windows
3. **Complex build process**: Custom dump mechanism, DOC file generation
4. **Many dependencies**: 20+ external libraries to detect

**Recommended approach**: Start with a minimal working build (core features only), then incrementally add options.
